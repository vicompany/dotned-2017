Actor Model

From Wikipedia:
“A mathematical model of concurrent computation that treats "actors" as the universal primitives of concurrent computation”

Populair frameworks
     

An actor:

  • Has an address
  • Has a state (which can be persisted)
  • Has a messagebox
  • Has single-threaded behaviour

Virtual Actor-model = Orleans

  • Actors are always present
  • Keybased instead of Uri based
  • Async methods instead of messagebox
  • Built on the .NET Framework

Microsoft Orleans basics

Terminology is slightly different

Grains are the actors

The classes you write to express the actor logic

The grains live in a silo

A process which hosts the grains

A bundle of silos are a cluster

Multiple processes, possibly across mutiple datacenters

Orleans features

  • Observers:
    The grain calls your outside the grain code
  • Timers & Reminders:
    The grain calls it self.
  • Persistence:
    Mutlple type of stores and per enviroment configurable
  • Management:
    Off the shelf cluster and silo management
